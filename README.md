## 这是什么

一个**离线、零网络依赖**的桌面小程序：在玩《明治维新：财阀（Nippon Zaibatsu）》时，把当前盘面状态录入助手，点一下「**求最优解**」，它会给出：

- 盖着朱印的**首选行动卡**；
- **推荐理由**（直接来自评估函数 `V(state)` 各项贡献）；
- **备选方案 Top-3** 及其评分差。

应用全部逻辑都在前端（HTML / CSS / JS + 内置决策引擎），打包后是一个**单文件 `.exe`**，双击即用，无需安装任何运行环境；也可以直接用浏览器打开调试。

## 核心特性

| | |
|---|---|
| 🎯 **行动推荐 + 解释** | 不是黑箱打分。每条推荐都附「为什么」——名次从第几升到第几、还剩几次结算、各评估项贡献多少。 |
| 🧠 **决策引擎** | `走子生成 → 模拟（执行 EffectSpec）→ 评估 V(state) → 搜索 → 解释`；单步贪心打分 + 对头部候选 **2 步前瞻精排**，缓解短视。 |
| 🃏 **60 张工厂效果** | §3 全部工厂效果（6 类型 × A/B × 编号 1–5）以 `EffectSpec` 结构编码，模拟器统一执行。 |
| 🗺️ **忠实盘面布局** | 海图视图、财阀板、契约墙、起始标记卡均用 CSS/SVG **还原同款布局与配色**，不嵌入受版权保护的美术素材。 |
| ⚖️ **决策偏好** | 稳健胜率 / 期望分数最大化 / 平衡，三种偏好动态调整评估权重。 |
| 🧩 **扩展模组** | 支持 2/3/4 人局、后期工厂模组（编号 5 工厂）、单人模式（Automa）。 |
| 🔌 **完全离线** | 仅用系统字体，无任何在线资源，`.exe` 双击即用。 |

## 决策引擎怎么取胜（要点）

- **取胜主路径——地区影响力名次分**：逐地区计算「我方影响力 + 火车（每列 +2，需该区有我标记） vs 各对手 vs 海外公司」的排名，按规则书三时期分值 `10/8/6/2 → 18/14/10/3 → 26/20/14/4` 给分，并对剩余结算次数加权。
- **评估 `V(state)`** 综合：地区名次分、三轨终局个人目标（`5/8/10 → 10/16/20` 阈值）、引擎产能、当前分、资源/合约折现、工人节奏与整顿成本，并按偏好动态调权。
- **搜索**：单步贪心 + 头部候选 2 步前瞻精排，提升取胜质量。

## 快速开始

### A. 直接用浏览器打开（任意系统，最快）
双击 [`zaibatsu_pkg/app/index.html`](zaibatsu_pkg/app/index.html)（Chrome / Edge 等现代浏览器即可）。

### B. 用启动器运行（带原生窗口）
```bash
cd zaibatsu_pkg
pip install pywebview      # 可选，缺失会自动回退到浏览器
python main.py
```

### C. 在 Windows 上打包成单文件 .exe
```bat
cd zaibatsu_pkg
build_exe.bat              # 自动装 pyinstaller/pywebview、清理、打包
```
完成后在 `dist\` 得到可执行文件，双击运行。等价手动命令：`pip install pyinstaller pywebview && pyinstaller app.spec`。

## 怎么用

1. **开局三步**：选人数（2/3/4 人）与位次、决策偏好（可选扩展模组）→ 初始化我方起始标记与数值 → 进入对局。
2. **录入状态**：左栏海图（雇用区 / 工人列 / 地区影响力 / 工厂池），右栏各玩家财阀板（步进器与增删列表维护工厂、工人、合同、好感）。
3. **求推荐**：点底栏「**我的回合 · 求最优解**」，右侧抽屉给出首选行动卡 + 理由 + 备选 Top-3。
4. 还支持：记录对手回合、回合重置、撤销、存档 / 读档（JSON）。

## 项目结构

```
zaibatsu_pkg/
├─ app/                 ← 应用本体（前端，全部逻辑）
│  ├─ index.html        ← 界面：三步开局向导 + 双栏对局界面 + 推荐抽屉
│  ├─ config.js         ← 规则配置：60 张工厂效果、影响力表、人数配置…
│  ├─ engine.js         ← 决策引擎：走子生成 → 模拟 → 评估 → 搜索 → 解释
│  └─ app.js            ← 界面控制器：录入、渲染、撤销、存档/读档
├─ main.py              ← 启动器（.exe 外壳）：本地静态服务 + 原生窗口
├─ app.spec / build_exe.bat / requirements.txt   ← 打包
└─ README.md            ← 打包与使用详解
```

> 应用全部功能都在 `app/` 前端代码里；`main.py` 只负责把它装进原生窗口，因此打包稳、离线跑，也方便直接用浏览器调试。详细的打包/运行/数值校对说明见 [`zaibatsu_pkg/README.md`](zaibatsu_pkg/README.md)。

## 关于游戏数据

设计文档 §9 列出的部分数值需对照实体卡牌校验，在 `app/config.js` 中以 `todo:true` 占位标注，**不影响引擎运行**——可在对局面板据实填写，校对实体卡后直接替换占位值即可，无需改动引擎。

## 版权说明

出于版权考虑，软件**不嵌入规则书/手册中的实际图片**，而是用 CSS/SVG 还原同款盘面布局、配色与图标排布，不复制受版权保护的美术素材。本项目为非官方的玩家辅助工具，与游戏出版方无隶属关系。
